# Contributing

Thanks for contributing to Moaiy.

## Before Opening a Pull Request

- Search existing issues/PRs to avoid duplicates
- Keep changes focused and small when possible
- Update docs if behavior changes
- Include tests for code paths that can be automated

## Pull Request Checklist

- Clear title and summary
- Linked issue (if applicable)
- Testing notes (what you ran, what passed)
- Screenshots/recordings for UI changes (if applicable)

## Commit Style

Repositories under this org prefer Conventional Commits:

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `refactor: ...`
- `test: ...`
- `chore: ...`

## Security

Do not open public issues for security vulnerabilities.
Use private reporting channels defined in `SECURITY.md`.
